\section{Related Work}

\paragraph{Transformers and efficient attention.}
The Transformer architecture replaces recurrence with self-attention, enabling parallel sequence modeling and forming the backbone of modern LLMs \citep{vaswani2017attention}. During decoding, attention over the growing prefix becomes expensive unless previous keys and values are cached. Systems and kernels such as FlashAttention and PagedAttention reduce memory traffic and improve serving efficiency without necessarily changing the model distribution \citep{dao2022flashattention,kwon2023efficient}. \uckv{} is complementary: it targets the semantic and reliability risk introduced when the cache itself is compressed.

\paragraph{KV cache eviction and compression.}
Several methods reduce KV memory by evicting or summarizing cache entries. Scissorhands relies on persistence of token importance during generation \citep{liu2023scissorhands}. H2O retains heavy-hitter tokens and recent tokens, framing eviction through a dynamic submodular perspective \citep{zhang2023h2o}. StreamingLLM preserves attention sinks together with a rolling cache to support stable streaming inference \citep{xiao2024streamingllm}. SnapKV uses attention patterns from an observation window to select prompt tokens before generation \citep{li2024snapkv}. PyramidKV allocates different budgets across layers based on a pyramidal information-funneling observation \citep{cai2024pyramidkv}. Ada-KV further studies adaptive head-wise budget allocation \citep{feng2024adakv}. These methods motivate our retention primitives, but they usually optimize quality-efficiency tradeoffs without making calibrated uncertainty a first-class control signal.

\paragraph{KV cache quantization.}
Quantization compresses cache activations rather than evicting tokens. KIVI studies asymmetric 2-bit KV quantization and observes different grouping preferences for keys and values \citep{liu2024kivi}. KVQuant pushes ultra-low-precision KV cache quantization with methods such as pre-RoPE key quantization and outlier handling \citep{hooper2024kvquant}. \uckv{} can be combined with quantization by applying risk-aware budget selection to the number of retained tokens and bit-width selection to retained activations.

\paragraph{Uncertainty estimation and calibration.}
Calibration asks whether a model's stated confidence matches its empirical correctness probability. Temperature scaling is a standard post-hoc calibration method for neural classifiers \citep{guo2017calibration}, while deep ensembles remain a strong general baseline for predictive uncertainty when multiple models are affordable \citep{lakshminarayanan2017deep}. In LLMs, uncertainty can be estimated from token probabilities, self-evaluation, multiple samples, semantic entropy, or dedicated confidence estimators \citep{kadavath2022language,farquhar2024detecting,geng2024survey}. Selective classification studies the risk-coverage tradeoff when a model abstains under high uncertainty \citep{geifman2017selective}. \uckv{} adapts this view from answer selection to memory selection: when cache compression risk is high, the system should abstain from aggressive compression.

\paragraph{Long-context evaluation.}
Long-context benchmarks reveal that large context windows do not guarantee reliable use of all input positions. LongBench evaluates multilingual and multitask long-context understanding \citep{bai2024longbench}. RULER provides configurable synthetic tasks beyond simple needle retrieval \citep{hsieh2024ruler}. Lost-in-the-middle analyses show that relevant information can be underused when placed in the middle of long contexts \citep{liu2024lost}. These findings are directly relevant to cache compression, since eviction can amplify position-dependent failures.
